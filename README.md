# Конфиг для Neovim 0.12+

Использует встроенные автодополнение и пакетный менеджер `vim.pack`, доступные с Neovim 0.12. Так же используется нативный **lsp**. Благодаря этому конфиг стал менее громоздким.

- Настройки, подключаемые в `init.lua`, и вспомогательные функции:

  ```sh
  lua
  ├── config
  │   ├── autocmds.lua # Общие автокоманды, например, восстановление курсора при входе
  │   ├── keymaps.lua  # Общие сочетания клавиш
  │   ├── options.lua  # Основные настройки редактора
  │   └── theme.lua    # Автозагрузка и сохранение темы, установленной через colorscheme, в theme.json
  └── utils.lua        # Вспомогательные функции
  ```

- `plugin/*.lua` — плагины и их настройки. Плагины устанавливаются и подключаются через `vim.pack.add`. У конифгураций плагинов адекватные имена файлов, те список сторонних цветовых схем находится в `colorschemes.lua`, а настройки **LSP** — в `lsp.lua`. Я не добавлял настройки **DAP**, так как дебаггером не пользуюсь, вместо этого генерирую тесты через нейронки. Тут минималистичный набор плагинов для веб-разработки (Python, PHP, Go, TS/JS, Vue.js...). Стандартный файловый менеджер я заменил `oil.nvim`, который позволяет редактировать список файлов в каталоге как текст, и тп. 

- `after/lsp/*.lua` — настройки языковых серверов, которые расширяют дефолтные, те те, что идут с `nvim-lspconfig`. Чтобы полностью предопределить настройки, их нужно класть в `lsp/` (у меня такой необходимости не возникало).

- Если нажать `<leader>` или `<C-w>`, то с помощью `whichkey` будут показаны все возможные связки клавиш (сочетания), начинающиеся с них. Используй `<leader>?`, чтобы посмотреть пользовательские сочетания.

- Для управления языковыми серверами и другими средствами разработки используй `:Mason`. Предустановленные языковые сервера перечислены в `ensure_installed` файла `plugin/lsp.lua`.

- Подсветка синтаксиса, фолдинг и пр. операции с кодом осуществлются через `tree-sitter` (встроенная поддержка). См. `TSInstall` (установка новых парсеров), `TSUpdate` и пр. Предустановленные парсеры перечислены в `treesitter.lua`, в `ensure_installed`.

Как использовать:

```sh
cd ~/.config
mv ~/.config/nvim{,.old}
git clone https://github.com/s3rgeym/nvim-config nvim
```
