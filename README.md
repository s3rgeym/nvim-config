# Конфигурация для Neovim 0.12+

Используются встроенное автодополнение и пакетный менеджер `vim.pack`, доступные начиная с версии Neovim 0.12. Также используется встроенный **LSP**. Благодаря этому конфигурация стала менее громоздкой.

- Настройки, подключаемые в `init.lua`:

  ```sh
  lua
  └── user
      ├── autocmds.lua # Общие автокоманды (например, восстановление позиции курсора)
      ├── keymaps.lua  # Общие сочетания клавиш
      ├── options.lua  # Основные настройки редактора
      └── theme.lua    # Автозагрузка и сохранение темы в theme.json
  ```

- `plugin/*.lua` — плагины и их настройки. Плагины устанавливаются и подключаются через `vim.pack.add`. У конфигураций плагинов понятные имена файлов: например, список сторонних цветовых схем находится в `colorschemes.lua`, а настройки **LSP** — в `lsp.lua`. Настройки **DAP** отсутствуют, так как я не использую отладчик, предпочитая генерацию тестов с помощью ИИ. Представлен минималистичный набор плагинов для веб-разработки (Python, PHP, Go, TS/JS, Vue.js и др.). Стандартный файловый менеджер заменён на `oil.nvim`, позволяющий редактировать структуру каталогов как обычный текст.

- `after/lsp/*.lua` — настройки языковых серверов, расширяющие стандартные параметры `nvim-lspconfig`. Для полного переопределения настроек их следует помещать в `lsp/` (на данный момент такой необходимости нет).

- При нажатии `<leader>` или `<C-w>` с помощью `which-key` отображаются все возможные сочетания клавиш, начинающиеся с этих префиксов. Используйте `<leader>?` для просмотра пользовательских привязок.

- Для управления языковыми серверами и другими инструментами разработки используйте команду `:Mason`. Список предустановленных серверов находится в параметре `ensure_installed` файла `plugin/lsp.lua`.

- Подсветка синтаксиса, сворачивание кода (folding) и другие операции осуществляются через **Tree-sitter** (встроенная поддержка). См. команды `TSInstall` (установка новых парсеров), `TSUpdate` и др. Список предустановленных парсеров находится в `plugin/treesitter.lua` в параметре `ensure_installed`.

### Как использовать:

```sh
# Резервное копирование старого конфига
mv ~/.config/nvim{,.old}
git clone https://github.com/s3rgeym/nvim-config ~/.config/nvim

# Либо использование этого конфига через NVIM_APPNAME
git clone https://github.com/s3rgeym/nvim-config ~/.config/s3rgeym-nvim
NVIM_APPNAME=s3rgeym-nvim nvim
```
