# Конфиг для Neovim 0.12+

Минималистичный и современный конфиг **Neovim**. Используются встроенное автодополнение и пакетный менеджер `vim.pack`, доступные начиная с версии 0.12. Также используется встроенный **LSP**. Благодаря этому конфиг стал менее громоздкой.

- Настройки, подключаемые в `init.lua`:

  ```sh
  lua
  └── config
      ├── autocmds.lua # Общие автокоманды (например, восстановление позиции курсора)
      ├── keymaps.lua  # Общие сочетания клавиш
      ├── options.lua  # Основные настройки редактора
      └── theme.lua    # Автозагрузка и сохранение темы в theme.json
  ```

- `after/plugin/*.lua` — плагины и их настройки. Плагины устанавливаются и
  подключаются через `vim.pack.add`. У конфигураций плагинов понятные имена файлов: например, список сторонних цветовых схем находится в `colorschemes.lua`, а настройки **LSP** — в `lsp.lua`. Представлен минимальный набор плагинов для веб-разработки (Python, PHP, Go, TS/JS, Vue.js и др.). Настройки **DAP** отсутствуют, так как я не использую отладчик, предпочитая генерацию тестов с помощью нейронок. Стандартный файловый менеджер заменён на `oil.nvim`, позволяющий редактировать структуру каталогов как обычный текст.

- `after/lsp/*.lua` — настройки языковых серверов, расширяющие стандартные параметры `nvim-lspconfig`. Для полного переопределения настроек их следует помещать в `lsp/` (на данный момент такой необходимости нет).

- При нажатии `<leader>` или `<C-w>` с помощью `which-key` отображаются все возможные сочетания клавиш, начинающиеся с этих префиксов. Используйте `<leader>?` для просмотра пользовательских привязок. Для избежания проблем не используются сочетания, когда одно сочетание является префиксом для другого (если есть сочетание `<leader>bd`, то вместо `<leader>b` лучше использовать `<leader>bb`).

- Для управления языковыми серверами и другими инструментами разработки используйте команду `Mason`. Список языковых серверов для установки находится в `plugin/lsp.lua`.

- Быстрая подсветка синтаксиса, сворачивание кода (folding) и другие операции осуществляются через **Tree-sitter** (встроенная поддержка). Языковые парсеры для автоматической установки перечислены в `plugin/treesitter.lua`, вручную их можно установить с помощью команды `TSUpdate`.

Как использовать:

```sh
# Резервное копирование старого конфига
mv ~/.config/nvim{,.old}
git clone https://github.com/s3rgeym/nvim-config ~/.config/nvim

# Либо использование этого конфига через NVIM_APPNAME
git clone https://github.com/s3rgeym/nvim-config ~/.config/s3rgeym-nvim
NVIM_APPNAME=s3rgeym-nvim nvim
```
