Минималистичный, быстрый и предсказуемый конфиг для **Neovim 0.12+**, построенный на нативных возможностях редактора.

Без внешних менеджеров плагинов. Без тяжёлых фреймворков. Без лишней магии.

Используется:

- `vim.pack` — встроенный менеджер пакетов
- Нативное автодополнение Neovim
- Встроенный LSP
- Tree-sitter

Конфигурация ориентирована на реальную ежедневную разработку, а не на коллекционирование плагинов.

- Настройки, подключаемые в `init.lua`:

  ```sh
  lua
  └── config
  ├── autocmds.lua # Общие автокоманды (например, восстановление позиции курсора)
  ├── keymaps.lua  # Общие сочетания клавиш
  ├── options.lua  # Основные настройки редактора
  └── theme.lua    # Автозагрузка и сохранение темы в theme.json
  ```

- `plugin/*.lua` — плагины и их настройки. Файлы загружаются стандартным
  механизмом Neovim, плагины устанавливаются через `vim.pack.add`. Имена
  файлов соответствуют назначению (например, `colorschemes.lua`, `lsp.lua`,
  `treesitter.lua`).

  Конфиг ориентирован на веб-разработку (Python, PHP, Go, TS/JS, Vue и др.) и
  включает минимально достаточный набор инструментов: LSP, форматирование,
  навигацию, git-интеграцию и работу с кодовой базой. **DAP** не используется.

  Встроенный файловый менеджер заменён на `oil.nvim`, который позволяет
  редактировать структуру каталогов как обычный буфер.

- `after/lsp/*.lua` — точечные настройки конкретных языковых серверов.
  Базовая конфигурация находится в `plugin/lsp.lua`, а в `after/lsp/`
  добавляются или переопределяются параметры отдельных серверов.

- При нажатии `<leader>` или `<C-w>` с помощью `which-key` отображаются все возможные сочетания клавиш, начинающиеся с этих префиксов. Используйте `<leader>?` для просмотра пользовательских привязок. Для избежания проблем не используются сочетания, когда одно сочетание является префиксом для другого (если есть сочетание `<leader>bd`, то вместо `<leader>b` лучше использовать `<leader>bb`).

- Для управления языковыми серверами и инструментами разработки используется
  `Mason`. Список серверов для установки определён в `plugin/lsp.lua`.

- Подсветка синтаксиса, folding и структурная навигация работают через
  встроенную поддержку **Tree-sitter**. Список парсеров для автоустановки
  указан в `plugin/treesitter.lua`. Обновление — командой `TSUpdate`.

## Установка

```sh
# Резервное копирование старого конфига
mv ~/.config/nvim{,.old}
git clone https://github.com/s3rgeym/nvim-config ~/.config/nvim

# Либо использование этого конфига через NVIM_APPNAME
git clone https://github.com/s3rgeym/nvim-config ~/.config/s3rgeym-nvim
NVIM_APPNAME=s3rgeym-nvim nvim
```

После первого запуска рекомендуется:

1. Установить LSP‑серверы и инструменты через `:Mason`
2. Установить все Tree‑sitter парсеры командой:

```vim
:TSInstall all
```

## Секреты

Для хранения локальных токенов и приватных ключей используется файл
`lua/config/secrets.lua`, который **не должен попадать в репозиторий**.

После клонирования создайте его из примера:

```sh
cp lua/secrets.lua.example lua/config/secrets.lua
```

И заполните необходимыми значениями. Файл добавлен в `.gitignore`.

## Структура

```
init.lua
lua/
config/        # базовые настройки (options, keymaps, autocmds, theme)
plugin/          # плагины и их конфигурация
after/lsp/       # точечные настройки языковых серверов
```

Базовая идея — простая и прозрачная архитектура. Всё разложено по смыслу, без скрытой динамической генерации конфигов.

## Для кого этот конфиг

- Для тех, кто хочет использовать нативные возможности Neovim
- Для веб‑разработки (Python, Go, TS/JS, Vue, PHP и др.)
- Для тех, кто ценит контроль над средой
- Для тех, кто не хочет зависеть от больших дистрибутивов

## Keymaps и Workflow

`<leader>` — основной префикс (пробел).

Используется `which-key`, поэтому при нажатии `<leader>` отображаются доступные действия.

Принципы:

- Нет конфликтующих префиксов
- Логичные группировки (buffers, git, lsp, search и т.д.)
- Минимум переопределения стандартных клавиш
- Работает независимо от раскладки (настроен `langmap`)

Типичный workflow:

1. Навигация по файлам через `oil.nvim` или `fzf`
2. Переход к определениям через LSP
3. Рефакторинг и переименование через LSP
4. Форматирование через `conform.nvim`
5. Поиск по проекту через `grug-far` или `fzf`

Конфиг ориентирован на быстрый переход от навигации к изменению кода без лишних панелей и всплывающих окон.

## Плагины

| Плагин                     | Назначение                                |
| -------------------------- | ----------------------------------------- |
| nvim-lspconfig             | Конфигурация LSP                          |
| mason.nvim                 | Управление LSP и инструментами            |
| nvim-treesitter            | Подсветка, folding, синтаксический анализ |
| conform.nvim               | Форматирование кода                       |
| gitsigns.nvim              | Git-интеграция в буфере                   |
| oil.nvim                   | Файловый менеджер как буфер               |
| fzf                        | Поиск файлов и строк                      |
| which-key.nvim             | Подсказки по сочетаниям клавиш            |
| lualine.nvim               | Statusline                                |
| indent-blankline.nvim      | Визуализация отступов                     |
| aerial.nvim                | Навигация по символам                     |
| undotree                   | История изменений                         |
| mini.surround / mini.pairs | Обрамление текста и автоматическая расстановка парных скобок |
| opencode.nvim              | Интеграция OpenCode прямо в Neovim        |

Список намеренно ограничен. Каждый плагин выполняет конкретную задачу и не дублирует встроенные возможности Neovim.

Для кого‑то даже этот список может показаться избыточным. Например,
**Tree-sitter** уже встроен в Neovim, а `nvim-treesitter` в конфиге
используется в первую очередь для удобной установки и обновления парсеров,
а не ради базовой подсветки.
